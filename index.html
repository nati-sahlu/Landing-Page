Introduction

Ray Casting

this is a 2D game which are developed using ray-casting technique.ray-casting is a techniques that transform a limited form of data(a very simplified map or floor plan) into a 3D projection by tracing rays from the view point inot the viewing volume. Ray-casting sensation began with the release of a game, Wolfenstein 3D(iD software), in 1992. raycasting can go very fast, because only a calculation has to be done for every vertical line of the screen.

Features

1.Orientation

In this part of the project, we have drawn a different color depending on the orientation of the walls. As you can see from the screen shoot, walls facing from one side is drawn in different collor from walls facing from the other side. some of the walls are lighter than the other. some are darker than the other walls

2.Rotation and Movement

Rotation In this part of the project, we provide a way to rotate the camera during the exectution. the camera is rotated using left & right keys.

and we have provided a way to move the camera during the execution.The camera is moved forward by pressing the up from the keyborad. on the other hand the camera is moved backward using down of the keyborad.

3.Collision

We detect collision of the player so that the player is not able to enter the walls. I have made the player slide on the walls instead of just stop it.simple change to some direction, then the player will slide on the walls. the application detects when the player approaches the wall. if the player is approaches the threshold value, it will be forced to slide the walls.

4.Multi-Task

move & rotate at the same time In this part of the project, I have added a way to move on several directions & rotate in the same time. for example you can move forwared & rotate in clockwise direction at the same time. This is one way of hadnling multiple events on the same frame

5.Rain Mode

The feature allows users to toggle a rain effect on and off by pressing the "R" key. When the key is pressed, the rain state switches between active and inactive, triggering or stopping a rain animation on

About Us

Our Maze Game was inspired by the timeless challenge of navigating through a labyrinth, testing both logic and problem-solving skills. Since we needed a project and didnâ€™t have an original idea at the time, we decided to take on the maze challenge, a classic problem in computer science. Mazes have been used in games and puzzles for centuries, and implementing one in C was an exciting opportunity to apply our programming skills. We wanted to create a program that could generate, display, and solve mazes, all while adhering to structured programming principles. Development Process & Timeline Project Selection: We opted for the Maze Game as our default project. Understanding Maze Algorithms: We explored different ways to generate and solve mazes, such as Depth-First Search (DFS) and Breadth-First Search (BFS). Coding the Logic: We wrote the game in C, focusing on efficient data structures like 2D arrays and linked lists for handling the maze grid. Testing & Debugging: We tested different maze sizes and paths to ensure the game was both functional and challenging.
